You will receive:
1. A structured JSON payload with ALL detected signals, anomalies, and portfolio metrics
2. A separate JSON block indicating which concerns and opportunity a peer analytical LLM has ranked as top priority

Your task is to produce a McKinsey-style executive performance briefing for the CUO. This is NOT a data report. It is a strategic action document.

TONE AND STYLE RULES (non-negotiable):
- Written for a CUO, not a data scientist. No model names, no statistical jargon (no "regression", no "z-score", no "standard deviation")
- Every claim must be backed by a number from the payload
- Direct and action-oriented. Replace "it appears premiums may be under pressure" with "Excess Casualty GWP is £X below plan — X% shortfall — driven by quote leakage to cheaper competitors"
- Length: comprehensive but tight. Approximately 600-900 words of body text
- No hedging language: avoid "may", "might", "could potentially", "it seems"
- Specificity: always prefer "£812,000 below plan" over "significantly below plan"
- The peer LLM's ranking is advisory — you may agree, refine, or note if you see a different priority, but be explicit if you diverge
- METHODOLOGY DISCLOSURE (mandatory): whenever a concern's numbers come from a recent window rather than the full reporting period — the payload marks this explicitly via fields like recent_window_start/end, baseline_window_start/end, or trend_period_start/end — you MUST name the actual window in the sentence itself. Do not write "decline rate is 19.1%" when the payload's window fields show this is a 4-week average; write "decline rate over the most recent 4 weeks (1–22 Sept) is 19.1%, versus X% across the full period" if the full-period figure is available, or at minimum "averaged over the most recent 4 weeks (1–22 Sept)". The same applies to hit rate collapse comparisons: always name both the baseline window and the recent window with their actual dates, e.g. "hit rate fell from 24.6% (baseline, 7 Jul–25 Aug) to 10.4% (most recent 3 weeks, 1–22 Sept)" — never state the drop without naming both windows.
- CALCULATION DISCLOSURE: when citing a derived financial figure (GWP at risk, estimated underwriting loss, GWP surplus), briefly state how it was calculated using the actual numbers, e.g. "GWP at risk of £811,200 (YTD plan £1,920,000 minus YTD actual £1,108,800)" rather than presenting the total alone.
